# White Salary Agent Runtime v2

## 目标

把 QQ、桌面端、工具、插件和后台任务接到同一套可恢复运行时，同时保留它们各自的交互规则。迁移采用旁路方式，不直接替换现有入口。

核心保证：

- QQ 群、QQ 私聊和桌面会话互相隔离。
- 工具运行期间可以继续接收消息、补充要求或取消任务。
- 工具结果重新交给主模型，由白按人设说明，不使用固定成功/失败话术。
- 跨端消息有持久队列、真实确认、重试和去重。
- 复杂任务可以并行拆分，但发消息、控制电脑等副作用必须串行。
- 现有工具、功能、插件、记忆、好感度和桌面语音能力逐项迁移，不整体重写。

## 架构边界

### ConversationActor

每个 `platform + scope + conversation_id` 对应一个串行 Actor。不同会话可以并行，同一会话的状态修改不会并发交错。

### TaskRuntime

任务状态：

`submitted -> working -> input_required/auth_required/cancel_requested -> completed/failed/cancelled`

不能安全中止的副作用工具进入 `cancel_requested`，直到工具返回真实结果后再进入终态，避免把仍在执行的操作谎报成已取消。

状态和事件写入 SQLite。进程重启后可以恢复未完成任务，不依赖内存队列。

### ToolLoop

保留现有 `tool_llm` 最终语义判断。工具候选先经过平台、权限、服务可用性和副作用过滤；执行阶段再次校验。工具结果作为事件返回主模型，允许继续调用其他工具。

### DeliveryBus

每次发送都有目标地址、幂等键、尝试次数、发送租约和平台回执。只有平台返回真实成功后才标记 `delivered`。如果平台可能已执行但回执丢失，状态进入 `unknown` 等待核对，绝不自动重复发送。

### MemoryScopes

- 用户长期记忆：跨平台共享。
- 会话短期上下文：按 QQ 群、QQ 私聊和桌面分别隔离。
- 群内用户私有状态：按 `group_id + user_id` 隔离。
- 任务上下文：按 `task_id` 保存，只向授权会话注入。

### EngagementLease

QQ群活动窗口按 `group_id + user_id` 管理。唤醒词、@白或引用白先建立短暂 pending 租约；只有回复真实发送成功或静默工具确认成功后才进入 engaged。相关续聊滑动续期，白提出问题时延长等待，有后台任务时持续保持；转头和别人说话会进入冷却，明确停话立即关闭。

## 迁移顺序

1. 新增运行时契约、SQLite 任务日志、会话 Actor、取消令牌和可靠 Outbox。
2. 增加用户级 QQ 活动租约，不改变旧 SmartReply 默认路径。
3. 在影子模式中同时运行新旧 QQ 决策，只记录差异，不重复发消息。
4. 接入工具循环和真实发送确认。
5. 接入 QQ 与桌面的双向持久投递。
6. 接入桌面语音状态、打断、字幕和后台任务完成通知。
7. 接入插件角色、工具元数据和记忆作用域。
8. 完成回放、故障注入和真实平台验收后再切换默认路径。

## 当前迁移状态（2026-07-12）

- 步骤 1、2、4、5、6、7 的基础模块和自动测试已经落地。
- 桌面 WebSocket 与 QQ/NapCat 已接入同一个 SQLite 任务账本；工具事件、取消和真实发送回执可追踪。
- 当前仍是 sidecar：原桌面/QQ 处理器继续决定回复、流式帧、插件和发送顺序，`AgentRuntimeService` 不接管实时入口。
- QQ 只有拿到真实 `message_id` 才完成任务；无回执进入待核对状态。桌面只有 `done` 帧发送成功才完成任务。
- 仍待完成真实平台回放、断网/重连/进程重启故障注入，以及“运行中补充要求”与 ConversationActor 的合并协议；这些验收通过前不切默认调度器。

## 非回归清单

迁移期间必须持续覆盖：

- QQ 唤醒词、引用、活动窗口、图文语音合并、表情包策略。
- 黑名单、白名单、好感度、隐私、休息和社交冷却。
- 独立 `tool_llm`、全部内置工具和插件提供的工具。
- 插件 observer/interceptor/rewriter/tool_provider 四类角色。
- 桌面长按语音、持续监听、TTS、字幕、Live2D、主动聊天。
- QQ、桌面、QZone、B站来源标记和跨平台长期记忆。
- 本地专用启动脚本和本地工具路径不随公开发布改动。

## 发布边界

本地工作区可能含机器专用路径和本地服务配置，不能原样推送公开仓库。公开版本只能从 `origin/main` 派生的独立工作树中移植经过审查的通用改动。
